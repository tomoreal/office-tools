# EDINET API仕様調査結果

## 概要
EDINET APIの主要機能を検証し、すべての機能が正常に動作することを確認しました。

## 実施日
2026-03-20

## APIキー
## APIキー
- APIキーは `.edinet_api_key_config` で管理（Git対象外）
- APIキーは全てのリクエストで`Subscription-Key`パラメータとして必要

## 検証したAPI

### 1. 書類一覧取得API
**エンドポイント**: `GET /api/v2/documents.json`

**パラメータ**:
- `date`: 対象日（YYYY-MM-DD形式）
- `type`: 2（メタデータを含む）
- `Subscription-Key`: APIキー

**結果**: ✅ 成功
- 指定日の提出書類一覧を取得可能
- 有価証券報告書は`docTypeCode='120'`でフィルタ可能
- XBRLフラグ（`xbrlFlag='1'`）で処理可能な書類を判定可能

**サンプルレスポンス**:
```json
{
  "metadata": {
    "resultset": {
      "count": 439
    }
  },
  "results": [
    {
      "filerName": "トヨタ自動車株式会社",
      "edinetCode": "E02144",
      "docID": "S100LO6W",
      "docTypeCode": "120",
      "submitDateTime": "2026-03-13 10:08",
      "xbrlFlag": "1"
    }
  ]
}
```

### 2. 企業検索機能
**実装方法**: 書類一覧APIを複数日分呼び出して企業名で検索

**パラメータ**:
- `company_name`: 検索する企業名（部分一致）
- `search_days`: 検索対象期間（デフォルト180日）

**結果**: ✅ 成功
- 企業名から`EDINETコード`を取得可能
- 複数企業がヒットした場合はリストで返却
- 7日間隔でサンプリングして効率化

**出力例**:
```
[1] トヨタ自動車株式会社
    EDINETコード: E02144
    証券コード: 72030
    最終提出: 2026-03-13 10:08
```

### 3. 有価証券報告書検索
**実装方法**: EDINETコードを指定して過去の有価証券報告書を検索

**パラメータ**:
- `edinet_code`: EDINETコード（例: E02144）
- `start_date`: 開始日（省略可）
- `end_date`: 終了日（省略可）

**結果**: ✅ 成功
- 指定企業の有価証券報告書を期間指定で取得可能
- 書類ID（`docID`）と対象期間を取得

**出力例**:
```
[1] 有価証券報告書－第117期(令和2年4月1日－令和3年3月31日)
    書類ID: S100LO6W
    提出日時: 2021-06-24 15:00
    対象期間: 2020-04-01 ～ 2021-03-31
```

### 4. 書類ダウンロードAPI
**エンドポイント**: `GET /api/v2/documents/{docID}`

**パラメータ**:
- `type`: ダウンロード種別
  - **1**: 提出本文書及び監査報告書（XBRL含む）← これを使用
  - 2: PDF
  - 3: 代替書面・添付文書
  - 4: 英文ファイル
  - 5: CSV形式のXBRL
- `Subscription-Key`: APIキー

**結果**: ✅ 成功
- ZIPファイルとしてダウンロード可能
- `type=1`でiXBRL形式のファイルが含まれる
- ファイルサイズ: 約1MB（トヨタ自動車の例）

**ZIPファイル構造**:
```
XBRL/
├── PublicDoc/
│   ├── *_ixbrl.htm (iXBRL形式の財務諸表)
│   ├── *.xsd (スキーマ定義)
│   ├── *_cal.xml (計算リンク)
│   ├── *_def.xml (定義リンク)
│   ├── *_lab.xml (ラベルリンク)
│   └── *_pre.xml (表示リンク)
└── AuditDoc/
    └── (監査報告書)
```

✅ EDINET APIの主要機能がすべて正常に動作
✅ 企業名検索 → EDINETコード特定 → 有価証券報告書検索 → XBRLダウンロードの一連のフローが実現可能

### 注意点
- EDINETコードリストAPIは利用困難（リダイレクトエラー）
  - **対策**: 書類一覧APIから企業を検索する方式を採用
- 企業検索は過去データからのサンプリングが必要
  - **効率化**: 7日間隔でサンプリング（180日分で26リクエスト）
- 土日祝日はデータがない場合があるため、エラーハンドリングが必要
